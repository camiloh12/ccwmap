## License notes

- **Public-domain sources** (NCES, IPEDS, FAA, GSA, HIFLD, USPS) carry no attribution requirement. We attribute them in this file for honesty but the app does not display per-pin source links for them.
- **GSA FRPP format + scope.** The public dataset is a single-sheet **`.xlsx`** (FY24 release is ~145 MB / ~308k rows), streamed read-only with openpyxl — not a CSV. State is filtered on the UPPERCASE `State Name` column (the `State Code` column is FIPS-numeric); the pin name comes from `Installation Name`. Only `Real Property Type == Building` is imported (Structures and Land are not "federal facilities" under 18 USC 930). Latitude/Longitude are present on ~97% of rows, so geocoding is a rare fallback.
- **GSA FRPP geocoding.** The few FRPP rows without coordinates are geocoded from their street address via the [US Census batch geocoder](https://geocoding.geo.census.gov/geocoder/) (`benchmark=Public_AR_Current`) — public domain, no API key, US-only. Rows the geocoder cannot match are dropped (never coordinate-faked) and counted in the dry-run report.
- **HIFLD Military → USACE MIRTA.** The legacy HIFLD Open hub was deactivated 2025-08-26. The military source now reads the USACE-owned MIRTA (Military Installations, Ranges, and Training Areas) feature service — "DoD Sites – Boundaries" (layer 1), polygon boundaries reduced to their centroid. Fields are UPPERCASE (`SITENAME`, stable GUID `SDSID`, `SITEREPORTINGCOMPONENT`). The pinned config URL is the layer's `/query?...&f=geojson` endpoint, which returns the full layer in one request (maxRecordCount 1000 > 825 features).
- **NCES K-12 (public schools), SY 2024-25.** Two files joined on `NCESSCH`: the EDGE public-school geocode file (`EDGE_GEOCODE_PUBLICSCH_2425.zip`, a `.zip` of an `.xlsx`; name, state, LAT/LON — the coordinate driver) + the CCD Common Core of Data preliminary nonfiscal directory bundle (`.zip` → `ccd_sch_029_*.csv`, `SY_STATUS` codes 1=Open/2=Closed/3=New/8=Reopened/4-7 inactive — operational filter, so closed campuses are excluded). Public schools only — private schools are not included. `fetch()` (via `importer/importer/sources/_archive.py`) flattens the EDGE xlsx to CSV and extracts the CCD directory member. ~16k candidates for TX/FL/PA. All candidates carry **native coordinates** (no Census geocoding or coordinate refinement).
- **IPEDS colleges, HD2024.** Single IPEDS HD ("Directory information") survey-year file (`HD2024.zip` → `HD2024.csv`, normalized to UTF-8) — no join; `UNITID` is the stable per-institution id (`INSTNM`, `STABBR`, `LATITUDE`, `LONGITUD`, `CYACTIVE`). Category `COLLEGE_UNIVERSITY`. Only FL has a `COLLEGE_UNIVERSITY` state-law cell; TX and PA candidates are emitted but drop at `apply_state_law` — these appear as "missing cells" in dry-run output. See `docs/importer/OMISSIONS.md` for the researched rationale. **Native coordinates** — no geocoding.
- **FAA commercial-service airports, CY2023 + 28-day NASR.** Two files joined on airport location id (`Locid` in the commercial-service list ↔ `ARPT_ID` in NASR `APT_BASE.csv`): the FAA "CY Enplanements at All Commercial Service Airports" workbook (a bare `.xlsx`; `Locid`, `ST`, `Airport Name`, `S/L`) provides the mixed-case name and the service filter, and NASR `APT_BASE.csv` (inside the 28-day `*_APT_CSV.zip`; `LAT_DECIMAL`, `LONG_DECIMAL` — signed decimal degrees) provides coordinates. The workbook is, by definition, the commercial-service set (≥2,500 annual enplanements); rows are kept where `S/L` ∈ {`P` primary, `CS` nonprimary commercial service} — these airports have TSA screening and a federal secured-area prohibition (`AIRPORT_SECURE`). General-aviation and other NPIAS public-use airports without commercial service are not in the list. Category `AIRPORT_SECURE` (federal-uniform `US` cell). ~60 candidates for TX/FL/PA. **Native coordinates** — no geocoding.
- **ODbL sources** (OSM) carry a share-alike obligation. After each apply run that lands OSM rows, the importer generates `dump-YYYY-MM-DD.csv.gz` (columns: `osm_type`, `osm_id`, `name`, `latitude`, `longitude`) and uploads it to the public Supabase Storage bucket `odbl-dumps` (created idempotently on first use). In-app attribution UI is deferred to Phase 7 (pilot ship). See below for the full OSM source details.
- **OSM via Overpass API.** Endpoint: `https://overpass-api.de/api/interpreter`. License: [ODbL](https://opendatacommons.org/licenses/odbl/) (share-alike). The `osm` source uses **per-state auto-scope**: for each requested state it inspects `states.yaml` and queries Overpass only for category/state combinations that have an `osm`-filtered cell — no query is ever sent for a category that would be dropped at `apply_state_law`. For the TX/FL/PA pilot that means `BAR_ALCOHOL` in TX and FL only; PA generates no Overpass query. Each query uses the `["ISO3166-2"="US-XX"]` area selector and `out center tags` to return element centroids. External-id format is `node/<id>`, `way/<id>`, or `relation/<id>`. Coordinate quality: **PRECISE** for nodes (OSM node coordinates); **BUILDING_POLYGON** (centroid of the way/relation bounding area) for ways and relations. After an apply run that lands OSM rows, the pipeline publishes `dump-YYYY-MM-DD.csv.gz` to the `odbl-dumps` Storage bucket (see ODbL note above).
- The work product we contribute — the state-law classifications applied on top of source pins — is dedicated to the public domain under CC0 (see [`data/state_laws/LICENSE`](../../data/state_laws/LICENSE)).
